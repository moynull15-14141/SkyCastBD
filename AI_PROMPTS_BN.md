## প্রম্পট ३.२: ইউজার রেজিস্ট্রেশন API

```
আমার Express সার্ভারে একটি সম্পূর্ণ ব্যবহারকারী রেজিস্ট্রেশন সিস্টেম তৈরি করে দাও।

চাই:
१. Models: User schema (MongoDB) - ইমেইল, পাসওয়ার্ড (bcrypt), নাম, তৈরির তারিখ
२. Routes:
   - POST /api/auth/register - নতুন ইউজার যোগ করা
   - POST /api/auth/login - লগইন এবং JWT টোকেন পাওয়া
   - GET /api/auth/profile - বর্তমান ব্যবহারকারী প্রোফাইল
   - POST /api/auth/logout - লগআউট

३. Middleware:
   - ইমেইল validation
   - পাসওয়ার্ড strength checking
   - JWT token authentication
   - Error handling

४. নিরাপত্তা:
   - bcrypt দিয়ে পাসওয়ার্ড হ্যাশ করা
   - JWT token (expires in 7 days)
   - Input sanitization
   - CORS setup

সম্পূর্ণ কোড দেবে প্রতিটি ফাইলের সাথে।
```

## প্রম্পট ३.३: প্রিয় শহর সংরক্ষণ API

```
আমার Express সার্ভারে প্রিয় শহর সংরক্ষণ এবং পরিচালনা করার জন্য API তৈরি করে দাও।

চাই:
१. Database Schema:
   - User ID (reference to User model)
   - City list (array): city name, latitude, longitude
   - Created/Updated timestamps

२. API Routes:
   - POST /api/favorites - নতুন শহর যোগ করা
   - GET /api/favorites - সব প্রিয় শহর দেখা
   - DELETE /api/favorites/:cityId - শহর সরানো
   - PUT /api/favorites/:cityId - শহর আপডেট করা

३. Validation:
   - প্রতি ইউজারে সর্বোচ্চ ১০টি শহর
   - ডুপ্লিকেট শহর চেক করা
   - Authenticated users শুধুমাত্র

४. সম্পূর্ণ কোড সহ error handling এবং response format।
```

३. যখন ঝড়ের সতর্কতা > জরুরি SMS

দাও:
१. Twilio API সেটআপ (Account SID, Auth Token পাওয়ার উপায়)
२. Express API routes:
   - POST /api/alerts/subscribe - মোবাইল নম্বর সাইন আপ
   - POST /api/alerts/send - সতর্কতা পাঠানো

   - প্রতি ৩০ মিনিটে চেক করা
   - শর্ত মেলে মিললে SMS পাঠানো

४. Database Schema: User phone numbers, alert preferences

५. সম্পূর্ণ কোড এবং খরচ অনুমান (মাসিক খরচ বলো)।
